# 📹 视频录制与提交指南

## 📱 录制设备选择

### 推荐方案
| 设备 | 优点 | 缺点 |
|------|------|------|
| **手机** | 便利、高清、方便编辑 | 需要支架固定 |
| **电脑摄像头** | 易操作、实时预览 | 需要屏幕录制软件 |
| **屏幕+手机** | 显示网页+硬件演示 | 需要两套设备 |

**最佳方案：** 手机 + 支架 + 屏幕镜像（可选）

---

## 🎥 录制前准备

### 环境准备
- [ ] 光线充足（自然光或台灯）
- [ ] 背景简洁
- [ ] 声音安静（录制时关闭风扇等干扰源）
- [ ] 手机/摄像头稳定固定
- [ ] 硬件在镜头范围内
- [ ] 网页浏览器能看清

### 物理准备
- [ ] LED 接好，能清晰看到亮度变化
- [ ] 触摸引脚可以轻松触摸
- [ ] 手机屏幕亮度适中
- [ ] ESP32 已成功连接 WiFi
- [ ] 网页已成功打开

### 测试录制
- [ ] 录制 30 秒测试视频
- [ ] 检查图像清晰度
- [ ] 检查音质（如需）
- [ ] 检查设备稳定性

## 🎬 每个实验的录制脚本

### Ex07 - Web 无极调光器（总时长：60-90秒）

**阶段1：介绍（5-10秒）**
```
镜头显示：
- ESP32 开发板
- 接线的 LED
- 手机屏幕上的网页

说话稿（可选）：
"这是 ESP32 Web 无极调光器实验。现在展示通过网页滑动条
实时控制 LED 亮度的效果。"
```

**阶段2：展示网页（5秒）**
```
镜头关键：
- 网页标题清晰可见
- 滑动条可见
- 当前亮度值显示

动作：
- 缓慢移动手机，让浏览器窗口更清晰
```

**阶段3：慢速拖动（20秒）**
```
动作序列：
1. 从左端（0亮度）缓慢拖到右端（255亮度）
   - 速度：约 10-15 秒完成
   - 观察 LED 从完全黑暗变到最亮
   - 网页上的数值从 0 变到 255

2. 暂停 2-3 秒，让观众看清最亮效果

镜头提示：
- 同时看到滑动条和 LED
- LED 亮度变化要清晰可见
```

**阶段4：快速调节（15-20秒）**
```
动作序列：
1. 快速在不同位置摆弄滑动条（20%-50%-80%)
2. 观察 LED 对应的亮度变化
3. 在中间亮度停留 2-3 秒

说话稿（可选）：
"无论如何拖动滑动条，LED 的亮度都能实时、平滑地跟随。"
```

**阶段5：总结（5秒）**
```
镜头：
- 再次显示整个系统：网页 + LED
- 可选：在网页显示"Ex07 Web 无极调光器"

说话稿：
"实验完成。LED 亮度通过网页成功实时控制。"
```

### Ex08 - 物联网安防报警器（总时长：90-120秒）

**阶段1：初始状态（5秒）**
```
镜头显示：
- 网页上的"布防"和"撤防"按钮
- 状态显示 "DISARMED"
- LED 处于熄灭状态

说话稿：
"这是物联网安防报警器。系统初始状态为撤防。"
```

**阶段2：点击布防（10秒）**
```
动作：
1. 用手指清楚地点击"布防"按钮（让手指在镜头中可见）
2. 等待网页更新
3. 观察状态变为 "ARMED"

说话稿：
"点击'布防'按钮，系统进入布防状态。"
```

**阶段3：触摸无反应演示（10秒）**
```
动作：
1. 系统处于 ARMED 状态
2. 用另一只手靠近/触摸触摸引脚
3. LED 无任何反应
4. 状态仍然显示 "ARMED"

说话稿：
"在布防状态下，触摸传感器暂时无反应。"
```

**阶段4：触发报警（15-20秒）**
```
动作：
1. 手指明确触摸触摸引脚（最好用食指，方便在视频中看清）
2. LED 立即开始快速闪烁（闪烁频率约 100ms）
3. 网页状态变为 "ALARM"
4. 保持手指接触 3-5 秒，让观众看清闪烁效果
5. 手指离开，LED 继续闪烁（继续报警）
6. 再靠近一次，确认系统锁定报警

说话稿：
"一旦触摸引脚，LED 立即进入高频闪烁的报警状态。即使
手指离开，LED 也会继续闪烁，直到用网页撤防。"

镜头关键：
- LED 闪烁要非常清晰
- 需要看到"ALARM"状态
- 需要看到手指的动作
```

**阶段5：撤防恢复（15秒）**
```
动作：
1. 点击"撤防"按钮
2. LED 立即熄灭
3. 网页状态变为 "DISARMED"
4. 即使此时触摸引脚，也无反应

说话稿：
"点击'撤防'按钮，系统立即恢复正常。LED 熄灭，
状态恢复为撤防。"
```

**阶段6：重复演示（20-30秒，可选）**
```
动作：
再演示一遍完整流程：
1. 布防 → 触摸 → 报警 → 撤防 → 熄灭

目的：
强化印象，确保关键现象清晰
```

**阶段7：总结（5秒）**
```
说话稿：
"实验完成。我们成功演示了一个简单的物联网安防系统。"
```

### Ex09 - 实时传感器 Web 仪表盘（总时长：60-90秒）

**阶段1：介绍网页（5-10秒）**
```
镜头显示：
- 网页标题 "ESP32 实时传感器 Web 仪表盘"
- 显示实时数值（一个大数字）
- 触摸引脚在镜头中可见

说话稿：
"这是实时传感器Web仪表盘实验。网页实时显示触摸
传感器的数值。"
```

**阶段2：手指靠近（20-30秒）**
```
动作序列：
1. 初始状态：数值显示某个值，如 "80"
2. 缓慢用手指靠近触摸引脚（约 5-10cm 距离）
3. 观察网页上的数值逐渐减小：80 → 60 → 40 → 20 → 10
4. 手指继续靠近至接近或接触引脚
5. 数值继续减小到最小值（如 2-5）
6. 保持手指接近 2-3 秒

说话稿：
"当我缓慢用手指靠近触摸传感器时，网页上的数值
实时、平滑地逐渐减小。"

镜头关键：
- 需要同时看到手指和网页数值
- 数值变化要清晰
- 可以来回靠近/离开多次
```

**阶段3：手指离开（15-20秒）**
```
动作：
1. 缓慢将手指移开（约 5-10cm 距离）
2. 观察数值逐渐恢复：10 → 20 → 40 → 60 → 80
3. 手指完全离开后，数值恢复到原始值
4. 暂停 2-3 秒

说话稿：
"当手指离开时，数值立即开始恢复。没有任何延迟。"
```

**阶段4：重复循环（20-30秒）**
```
动作：
再做 2-3 遍 "靠近 → 离开" 的循环：
1. 靠近（5-10秒）
2. 离开（5-10秒）
3. 再靠近...

目的：
展示系统的连贯性和实时性
```

**阶段5：快速变化演示（10秒，可选）**
```
动作：
快速摇晃手指在引脚附近，让数值快速变化

说话稿：
"系统反应灵敏，能够捕捉到快速的变化。"
```

**阶段6：总结（5秒）**
```
说话稿：
"实验完成。我们成功演示了一个实时的传感器
数据监控系统。"
```

## 🎨 视频编辑要点

### 添加片头（5-10秒）

创建一个简单的片头包含：
```
[1秒] 黑屏 + 音乐或音效
[3-5秒] 显示：
        实验编号：Ex07 / Ex08 / Ex09
        实验名称：...
        项目名称：ESP32 物联网实验
[1秒] 过渡到主要内容
```

**工具推荐：**
- 手机：剪映、CapCut
- 电脑：OBS Studio（免费）、Camtasia

---

### 添加文字说明

在关键时刻添加字幕，解释正在发生的事情：

**Ex07 字幕示例：**
```
[15秒] "拖动滑动条"（用箭头指向滑动条）
[35秒] "LED 亮度实时变化"（用箭头指向 LED）
[55秒] "完全亮度"（突出显示数值 255）
```

**Ex08 字幕示例：**
```
[5秒] "初始状态：DISARMED"
[15秒] "点击布防"
[25秒] "系统状态：ARMED"
[35秒] "触摸引脚"
[40秒] "报警：LED 高频闪烁"
[60秒] "点击撤防"
[70秒] "LED 立即熄灭"
```

**Ex09 字幕示例：**
```
[10秒] "初始数值：80"
[20秒] "手指靠近"
[30秒] "数值逐渐减小：20"
[40秒] "数值持续减小：5"
[50秒] "手指离开"
[60秒] "数值恢复：80"
```

---

### 添加背景音乐

- 不要太吵闹（背景音乐为主）
- 避免版权问题（使用免费素材库）
- 音乐时长应与视频相同

**免费素材库：**
- YouTube 音频库
- Freepik Music
- Pixabay Music
- Pexels Music

## 📤 视频格式与规格

### 推荐规格
```
分辨率：1080p (1920×1080) 或 720p (1280×720)
格式：MP4
视频编码：H.264
音频编码：AAC
帧率：30fps 或 60fps
比特率：5-10 Mbps
音量：-3dB（略低于最大值）
```

### 文件大小
- 60 秒 1080p：约 50-100 MB
- 60 秒 720p：约 20-50 MB

## 💡 提示与建议

✅ **做**
- 确保光线充足，硬件清晰可见
- 动作缓慢，让观众有时间理解
- 多做几遍，选择最好的一次
- 展示完整的流程
- 添加清晰的文字说明

❌ **不要**
- 不要用摇晃的手机录制
- 不要用过快的播放速度
- 不要在黑暗环境中录制
- 不要忽视音质问题
- 不要只做单一角度，适度改变机位
