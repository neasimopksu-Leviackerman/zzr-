# Ex03 演示说明

## 项目演示内容

本目录包含 Arduino SOS 闪烁信号的实现，展示使用 `millis()` 函数进行精确定时控制。

## 硬件演示要求

### 硬件连接
```
Arduino Uno
    |
    Pin 13 ──[220Ω 电阻]── LED 负极 ──┐
                                       │
                                    GND (0V)
    
    5V ──────────── LED 正极
```

## 四个版本说明

### 1. 原始版本 (sos_signal.ino)
- **特点**：直接的顺序逻辑
- **输出**：基本的状态信息
- **用途**：理解基本的 SOS 实现

**运行效果**：
```
[0ms] Pause End, Starting new SOS
[100ms] Flash: ·
[200ms] Flash: ·
[300ms] Flash: ·
[500ms] Flash: —
...
```

### 2. 改进版本 (sos_signal_enhanced.ino)
- **特点**：清晰的代码结构，详细的日志
- **输出**：每个周期的完整信息
- **用途**：验证信号时序正确性

**运行效果**：
```
[0ms] SOS Cycle #1 Complete
[1000ms] ===== SOS #1 Complete =====
```

### 3. 状态机版本 (sos_signal_statemachine.ino)
- **特点**：使用状态机设计模式
- **优势**：代码更模块化，易于维护和扩展
- **输出**：符号级别的详细跟踪

**运行效果**：
```
[100ms] Flash #1 | Symbol 1/9 [S]: · (100ms)
[200ms] Flash #2 | Symbol 2/9 [S]: · (100ms)
...
```

### 4. 可编程版本 (sos_signal_programmable.ino)
- **特点**：完整的摩尔斯码库，易于添加自定义消息
- **优势**：可扩展，支持多种摩尔斯码
- **输出**：完整的摩尔斯码信息

**运行效果**：
```
[100ms] Morse: S (1/3) -> · (100ms)
[200ms] Morse: S (2/3) -> · (100ms)
...
[5000ms - System Uptime: 5s] ===== SOS #1 Complete =====
```

## 技术细节

### millis() 的优势
在 Ex03 中，使用 `millis()` 而非 `delay()` 的主要原因：

1. **非阻塞**：程序不会停下来等待
2. **并行**：可以同时处理多个定时任务
3. **精确**：基于系统时钟，精度更高
4. **可控**：可以随时检查和响应输入

### SOS 信号的摩尔斯码

| 字母 | 代码 | 含义 |
|------|------|------|
| S | · · · | Three dots |
| O | — — — | Three dashes |
| S | · · · | Three dots |

**总时长**：
- 短闪（·）：100ms
- 长闪（—）：500ms  
- 间隔：100-300ms
- 停顿：1000ms

**完整周期时长**：
```
S(300+100) + O(1500+100) + S(300+100) + 停顿(1000) = 3400ms
实际：约 4.8 秒/周期（包含各间隔）
```

## 常见问题

### Q: 为什么要选择 LED 与电阻连接？
**A**: 220Ω 电阻的作用是限流，保护 Arduino 输出引脚和 LED。

### Q: 为什么 SOS 信号这样设计？
**A**: 这是国际公认的求救信号，在摩尔斯码通信中使用。

### Q: 可以用其他引脚吗？
**A**: 可以，修改代码中的 `pinMode(13, OUTPUT)` 中的数字即可。

### Q: 如何添加新的摩尔斯码消息？
**A**: 在可编程版本中，修改 `sos_message[]` 数组，参考注释中的例子。
